Step 1: Initiating a Research Job

To start a job, send a `POST` request to `/api/v1/research` with your query and file URLs. For security, we only accept pre-signed GET URLs, ensuring you retain full data custody. You can also provide an optional `callbackUrl` for webhooks and `metadata` for your own tracking.

Request Body:

{ "query": "Analyze these documents for evidence of medical malpractice...", "files": [ { "fileName": "Exhibit A - Dr. Smith Notes.pdf", "downloadUrl": "https://client-storage.com/path/to/file1?signature=..." } ], "callbackUrl": "https://api.your-app.com/webhook/depogenius-results", "metadata": { "internalCaseId": "case-12345" } }

Immediate Response:

{ "jobId": "eager-research-job-xyz789", "status": "queued", "statusUrl": "https://api.depogenius.com/api/v1/research/eager-research-job-xyz789" }

Step 2: Processing Your Documents

Our proprietary Clarisearch™ pipeline processes your documents using patent-pending technology that achieves fixed-cost analysis at scale. Files are securely downloaded from your temporary URLs and analyzed by our AI models. The system synthesizes all relevant findings into the structured Evidence Packet. Once processing is complete, source documents are permanently and irrecoverably deleted from our environment.

Polling for Progress (Optional):

You can make a `GET` request to the `statusUrl` to check on a job's progress.

{ "jobId": "eager-research-job-xyz789", "status": "processing", "progress": { "totalFiles": 2, "completedFiles": 1, "currentFile": "Deposition of Jane Doe.txt" } }

Step 3: Receiving Your Results

Once the job is complete, we notify you. We recommend using webhooks for efficiency. If you provided a `callbackUrl`, our server sends a `POST` request to it with the job status and a secure, pre-signed URL to download your JSON Evidence Packet.

Webhook Payload:

{ "jobId": "eager-research-job-xyz789", "status": "completed", "resultsUrl": "https://storage.googleapis.com/depogenius-results/final.json?sig=...", "metadata": { "internalCaseId": "case-12345" } }

Polling for Completion:

If polling, the status response will update once complete.

{ "jobId": "eager-research-job-xyz789", "status": "completed", "resultsUrl": "https://storage.googleapis.com/depogenius-results/final.json?sig=..." }

Getting Started

1. Obtain Your API Key

To use the Clarisearch™ API, you'll need an API key. Contact our team to get your unique API key, which will have the format csk_xxxxxxxxx.

Security Note: Store your API key securely and never expose it in client-side code. All API requests must be made from your server environment.

2. Secure Your Files (Critical)

Clarisearch™ uses a zero-trust security model. Your files remain in your own secure storage, and you provide temporary pre-signed URLs for access. We never request permanent credentials or ask you to make files public.

✅ Secure Approach: Pre-Signed URLs
  • • Files remain private in your storage system
  • • URLs are time-limited (4+ hours recommended)
  • • Each URL grants access to exactly one file
  • • Complete audit trail and control
❌ Never Do This:
  • • Don't make files publicly accessible
  • • Don't share permanent storage credentials
  • • Don't use URLs that expire too quickly

Quick Example (Google Cloud Storage):

// Generate secure, time-limited URL const [signedUrl] = await storage .bucket('your-private-bucket') .file('sensitive-document.pdf') .getSignedUrl({ version: 'v4', action: 'read', expires: Date.now() + 6 * 60 * 60 * 1000 // 6 hours }); // Use in API call const files = [{ fileName: 'sensitive-document.pdf', downloadUrl: signedUrl // ✅ Secure, time-limited }];

See the complete security guide with examples for all major cloud providers below.

Polling Best Practices

Efficient Status Monitoring

1. Use ETags for Efficient Polling

Always include the If-None-Match header with the ETag from your previous response. This allows the server to return a 304 Not Modified when nothing has changed.

// Example: Efficient polling with ETags let etag = null; async function checkJobStatus(jobId) { const headers = { 'Authorization': 'Bearer ' + apiKey }; if (etag) { headers['If-None-Match'] = etag; } const response = await fetch(statusUrl, { headers }); if (response.status === 304) { // Job hasn't changed, use cached data return previousJobData; } if (response.status === 200) { etag = response.headers.get('ETag'); return await response.json(); } }
2. Follow Adaptive Polling Intervals

Use the nextRecommendedPollInSeconds value from the API response to optimize your polling frequency and avoid unnecessary requests.

Active Processing:

3-30 seconds

Queued:

60-600 seconds

Terminal States:

Stop polling

3. Handle Rate Limits Gracefully

If you receive a 429 Too Many Requests response, check the Retry-After header and wait the specified number of seconds before retrying.

Clarisearch™ implements a zero-trust security model. We never request permanent access to your storage systems or ask you to make files publicly accessible. Your sensitive documents remain in your control at all times.

✅ How It Works
  • 1. You generate temporary, pre-signed URLs

  • 4. Clarisearch downloads and processes files
  • 5. Original files remain in your private storage
  • 6. You maintain complete audit trail

  • • No permanent credential sharing
  • • Time-limited access automatically expires

const { Storage } = require('@google-cloud/storage'); const storage = new Storage({ projectId: 'your-project-id', keyFilename: 'service-account-key.json' }); async function generateSecureUrls(files) { const secureFiles = []; for (const file of files) { const [signedUrl] = await storage .bucket('your-private-bucket') .file(file.path) .getSignedUrl({ version: 'v4', action: 'read', expires: Date.now() + 6 * 60 * 60 * 1000, // 6 hours }); secureFiles.push({ fileName: file.name, downloadUrl: signedUrl }); } return secureFiles; }

Automatic Evidence Transformation

When evidence packets are retrieved from Clarisearch, the DepoGenius system automatically transforms the evidence structure:

  • • Parses clientReference to extract DepoGenius IDs and source types
  • • Replaces Clarisearch internal file_id with DepoGenius file_id
  • • Maps source types to file_caseSource values (e.g., "CaseFile" → "case_file")
  • • Preserves file_title for user-friendly display
  • • Maintains client_reference for debugging and audit purposes

Evidence Packet Structure (Post-Transformation)

After transformation, evidence items in the packet have the following key fields for DepoGenius integration:

{ "evidence_id": "evidence-workflowId-1", "file_id": "abc123xyz", // DepoGenius file ID (transformed) "file_caseSource": "case_file", // Mapped from sourceType "file_title": "Contract Agreement.pdf", // From original fileName "client_reference": "depogenius:file:abc123xyz:CaseFile", // Preserved for debugging "snippet": "The parties agree to...", "relevance_score": 0.95, "relevance_explanation": "...", // ... other evidence fields }

Note: The transformation happens automatically in the clarisearchJobPoller function when retrieving completed jobs. No additional client-side processing is required.
